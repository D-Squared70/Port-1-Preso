    Chapter 05

    What · Why · How

    Fix the prompt before you blame Claude
    W
    What
    what you want it to do and give you back
    W
    Why
    the intent (people often skip)
    H
    How
    the constraints
    For a recurring task that must stay high-quality
    WWH base prompt
    AI improves it

    Look up current best practices for prompting Opus 4.8 as of [today's date]. Then rewrite my prompt in a targeted way — information-dense and no longer than it needs to be. Give me the improved prompt only.

    Two lines to paste at the end of any document prompt
    Grounding
    "Base your answers only on what's in the file I gave you."
    No Guessing
    "If the file doesn't contain enough to answer, say so rather than guessing."

    Chapter 07

    The Skill Loop

    Build one only when all three are yes
    Will I do this again?
    Does quality matter each time?
    Does it cut across topics / clients?
    Never skill an unproven task
    01
    Map
    Rant out the rough shape.
    02
    Prove
    Run it for real until it's good.
    03
    Capture
    Now turn it into a skill.
    04
    Patch
    Fix recurring misses.
    Write checks as binary, not vague
    VagueBinary — pass or fail, no judgment
    "Make it professional." "Every action item has an owner named — if not, write 'owner not specified'."
    "Keep it short." "Under 150 words."
    "Cite your sources." "Every figure has a document name + page number."
